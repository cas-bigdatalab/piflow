# 数据集检索技能（scienceDB + InstDB）

## Overview

当用户询问“我要 xxx 相关数据集”、“帮我找 xxx 的数据集”等与 scienceDB 或 InstDB 数据集检索相关的问题时，
使用本技能调用 `scidb_search_main` 与 `instDB_search_main` 完成检索、匹配评估与分类输出。

## Instructions

### 1) 触发条件与意图解析
- **语言对齐**：如果用户输入中文，生成的检索关键词必须为**中文**；如果用户输入英文，关键词必须为**英文**。严禁混合语言。
- **意图解析**：识别主题/领域、核心实体、时间范围（注意：当前年份为2026年，“近20年”指2006~2026，但**不要**将“近20年”作为检索词）、数据类型、用途。
- **关键约束检查**：如果用户需求过泛（如“找生物数据”）或存在冲突，先提出 1 个澄清问题。否则直接进行下一步。

请执行以下逻辑：


### 2) 关键词生成策略（核心优化点）
**禁止**直接从用户句子中截取词语。你必须进行**语义重组**，生成 3 个高价值的检索词。
请遵循以下思维链（CoT）：
1.  **分析核心概念**：用户说的“倒装焊”在学术数据库中可能被称为“Flip Chip Bonding”、“C4 Bumping”或“倒装封装”。
2.  **扩展同义/上位词**：思考该领域常用的专业术语、缩写或相关工艺名称。
3.  **组合策略**：
    - 关键词 1：**核心专业术语**（最精准的学术名称，如“倒装焊缺陷检测”而非“倒装焊”）。
    - 关键词 2：**应用场景 + 对象**（如“芯片封装 空洞数据”而非“芯片 数据”）。
    - 关键词 3：**相关技术/方法**（如“X射线 焊点 数据集”而非“X射线”）。

**❌ 错误示范（机械切分）：**
用户：“我要找倒装焊相关数据集”
生成：["倒装", "焊", "数据集"] 或 ["倒装焊", "相关", "数据集"] -> **禁止这样做！**

**✅ 正确示范（语义重组）：**
用户：“我要找倒装焊相关数据集”
生成：["倒装焊缺陷检测", "倒装焊接", "焊球空洞 X射线数据"] (根据用户语言选择)

### 3) 调用工具 `scidb_search_main` 和 `instDB_search_main`

核心目标：
- 基于生成的 3 个关键词，**逐个**调用工具检索。
- **第一轮**：使用初始生成的 3 个关键词分别调用。
- **结果评估**：检查返回结果的标题/描述是否真正命中用户的核心需求（不仅仅是包含关键词，而是语义相关）。
- **第二轮（仅当第一轮效果不佳时）**：
    - 若结果不相关，调整关键词策略（例如：从“宽泛术语”调整为“具体工艺参数”或“特定缺陷类型”）。
    - 使用调整后的新关键词再次调用（每个工具最多两轮：初始 + 1 次调整）。
- **调用规范**：
    - 每次调用仅使用**一个**关键词字符串。
    - `scidb_search_main` 参数：`keyword` (string), `size` (int)。
    - `instDB_search_main` 参数：`keyword` (string)。
    - 来源标注：`scidb_search_main` 结果标记为 `scienceDB`，`instDB_search_main` 结果标记为 `InstDB`。

### 4) 结果判定与输出格式

结果判定：
- 必须命中核心主题或关键实体。
- 若用户有明确的时间/地域/格式约束，结果需显式满足，否则视为不匹配。
- 若两轮检索后仍无匹配结果，明确说明“未在库中找到高度相关的数据集”

输出格式（分类展示）：
  - 数据集名称：标题或中文名称
  - 大小
  - 来源
  - 下载链接
